# iRacing Engineer

I have been thinking about how to use LLMs to help with automation. In my experience with LLMs, the technology excels at _decision making_. and one idea I had was to use it to help me with iRacing. I both drive AND (sometimes) run a live stream of my races. In both cases I see potential for LLMs to offload some of the cognitive load from myself.

- **iRacing** - When driving there is a lot of information to keep track of. In real life, drivers have whole teams monitoring the car and the competition. In iRacing, that falls on the driver alone. My idea here is to feed iRacing Telemetry to an LLM and have it make decisions about the information. For example, looking at fuel consumption and making decisions about when to pit, then feeding that information back to the driver as audio cues.
-
- **Live Streaming** - When streaming, there is a lot of information to keep track of. Some of this can be scripted, but there are times where having a system which can make decisions on the fly would be helpful. For example, when I am streaming, I may want to show the stats panel to the audience. Currently I have to manually toggle this via a button in my racing setup. I think the LLM could be fed information about the race and make a decision when to show this information.
