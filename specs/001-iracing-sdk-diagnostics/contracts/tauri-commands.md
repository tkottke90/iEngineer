# Contract: Tauri Commands

**Feature**: `001-iracing-sdk-diagnostics` | **Date**: 2026-06-26

These are the `invoke()` commands exposed by the Rust backend and called by the
Preact frontend via `@tauri-apps/api/core`. All types are generated by `ts-rs`
and available in the TypeScript bindings.

---

## `get_iracing_status`

Returns the current iRacing connection status without waiting for a state change.
Used to initialise the UI on page load before the first event fires.

```typescript
import { invoke } from '@tauri-apps/api/core';

const status: ConnectionStatus = await invoke('get_iracing_status');
// ConnectionStatus = 'Connected' | 'Disconnected' | 'Connecting'
```

**Rust signature**:
```rust
#[tauri::command]
pub async fn get_iracing_status(state: State<'_, AppState>) -> Result<ConnectionStatus, String>
```

**Error cases**: None expected — returns `Disconnected` if state is unavailable.

## `get_session_data`

Returns the current session metadata, or `null` if connected but no session is
active (iRacing is on the main menu).

```typescript
const session: SessionInfo | null = await invoke('get_session_data');

interface SessionInfo {
  track_name: string;
  session_type: string;   // "Race" | "Practice" | "Qualify" | ...
  car_name: string;
  wall_clock_time: string; // "HH:MM:SS"
}
```

**Rust signature**:
```rust
#[tauri::command]
pub async fn get_session_data(state: State<'_, AppState>) -> Result<Option<SessionInfo>, String>
```

**Error cases**: Returns `Ok(None)` when connected but no session loaded. Returns
`Err` only if the state lock is poisoned (should not occur in normal operation).

## `list_telemetry_fields`

Returns the full list of telemetry fields available in the current session, with
their current instantaneous values. Returns an empty list if not connected.

```typescript
const fields: TelemetryField[] = await invoke('list_telemetry_fields');

interface TelemetryField {
  name: string;           // canonical SDK variable name, e.g. "Speed"
  description: string;    // human-readable, e.g. "Velocity of car in world frame of reference"
  unit: string;           // e.g. "m/s", "rpm", "%", ""
  var_type: VarType;
  value: TelemetryValue;
}

type VarType = 'Char' | 'Bool' | 'Int' | 'Bitfield' | 'Float' | 'Double';

type TelemetryValue =
  | { Float: number }
  | { Double: number }
  | { Int: number }
  | { Bool: boolean }
  | { Bitfield: number }
  | { Char: string }
  | { FloatArray: number[] }
  | { IntArray: number[] }
  | 'Unavailable';
```

**Rust signature**:
```rust
#[tauri::command]
pub async fn list_telemetry_fields(state: State<'_, AppState>) -> Result<Vec<TelemetryField>, String>
```

**Notes**:
- Returns the cached `field_cache`. Values are as of the last 10 Hz tick.
- Fields not in the watchlist will have stale values — acceptable for the browser list.
- Call once on connection; the `iracing://session-changed` event signals when to
  call again (new session may expose different fields).

## `get_watchlist`

Returns the list of field names currently in the user's watchlist.

```typescript
const watchlist: string[] = await invoke('get_watchlist');
// e.g. ["Speed", "RPM", "Throttle", "Brake", "Gear"]
```

**Rust signature**:
```rust
#[tauri::command]
pub async fn get_watchlist(state: State<'_, AppState>) -> Result<Vec<String>, String>
```

---

## `set_watchlist`

Replaces the watchlist with the provided list of field names. The Rust polling
task uses this on the next 10 Hz tick. Invalid field names are silently ignored
(they will appear as `Unavailable` in tick events).

```typescript
await invoke('set_watchlist', { fields: ['Speed', 'RPM', 'Gear'] });
```

**Rust signature**:
```rust
#[tauri::command]
pub async fn set_watchlist(
    fields: Vec<String>,
    state: State<'_, AppState>,
) -> Result<(), String>
```

**Notes**:
- Replaces the entire watchlist (not incremental add/remove). The frontend manages
  the set locally and calls `set_watchlist` on any change.
- Order of fields in the list is preserved in tick event payloads.
